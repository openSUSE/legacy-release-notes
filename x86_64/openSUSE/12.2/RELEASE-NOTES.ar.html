3. عام

3.1. وثائق أوبن سوزي

  • في البدء، تجد إرشادات التثبيت خطوة بخطوة، وكذلك مقدمات لسطح المكتب كيدي وجنوم وليبر أوفيس. وأيضا المواضيع الأساسية للإدارة مثل نشر وإدارة البرامج، ومقدمة للتعرف على الشل.

  • مرجع يغطي الإدارة، وتكوين النظام بالتفصيل، ويشرح كيفية إعداد مختلف خدمات شبكة الاتصال.

  • يقدم دليل الأمن المفاهيم الأساسية لنظام الأمن، تغطي الأمن المحلي وأمن الشبكة.

  • تحليل النظام ودليل الضبط مع اكتشاف المشاكل وتقديم الحل الأمثل.

  • المحاكاة الافتراضية تقدم KVM مقدمة لإعداد وإدارة المحاكاة الافتراضية مع أدوات KVM, libvirt QEMU.

3.2. Pre-installation Memory Test Incorrectly Identifies Good Memory as Bad

The pre-installation memory test (memtest) on the openSUSE 12.2 media got miscompiled. It reports errors in test 7 on good RAM modules. Use the openSUSE 12.1 media if you need to run memtest.

4. ترقية النظام

4.1. Remote Update via "zypper dup"

When upgrading from openSUSE 12.1 (or older), openSSH connections will be closed when the new openssh package is upgraded. If you are upgrading with "zypper dup" over SSH, run "zypper dup" inside a resumable terminal multiplexer (e.g., "screen" or "tmux") so that you can re-connect easily, or at least immune to connection loss (e.g., via "nohup").

4.2. sysvinit تم تجاهله

بعض مكونات سطح المكتب تعتمد على الخدمات التي يقدمها systemd فقط. لذلك لا يزال أوبن سوزي 12.2 يدعم الإقلاع الأساسي للنظام مع sysvinit كاحتياطي، sysvinit ومع ذلك يعتبر مهمل وربما تسبب في خلل أو كسر لهذا السبب. إذا كان لديك أي مشاكل مع الإقلاع بنظام sysvinit ، جرب استخدام systemd قبل الإبلاغ عن تقارير الأخطاء.

4.3. توصيل losetup قليل الدعم من أجل cryptoloop

cryptoloop يعرف بالضعف ويعتبر قديم وهو صالح مع dm-crypt منذ سنوات. توصيل (مثلاً، عن طريق /etc/fstab) and losetup الآن يدعم cryptoloop. وهذا يعني أن إدخالات fstab القديمة التي تستخدم cryptoloop للوصول إلى المحتويات المشفرة لم تعد تعمل بهذه الطريقة. لا زالت إمكانية الوصول للحاويات باستخدام dm-crypt (/etc/crypttab), بالرغم من ذلك. الرجوع إلى http://en.opensuse.org/Encrypted_Filesystems للأمثلة على كيفية استخدامها بأسلوب جديد.

4.4. ضم الأقسام المشفرة مع systemd

إذا لم يتم تحميله الأقسام المشفرة تلقائياً مع systemd، يمكن أن يكون في هذه الحالة عُلِم noauto في /etc/fstab لهذه الأقسام. استبدال هذه العلامة ب nofail ليتم إصلاحها. على سبيل المثال، تغيير السطر التالي:

/dev/mapper/cr_sda3  /home   ext4    acl،user_xattr،noauto 0 2

إلى

/dev/mapper/cr_sda3  /home   ext4    acl،user_xattr،nofail 0 2

5. تقني

5.1. تهيئة الرسومات مع KMS (إعداد وضع النواة)

With openSUSE 11.3 we switched to KMS (Kernel Mode Setting) for Intel, ATI and NVIDIA graphics, which now is our default. If you encounter problems with the KMS driver support (intel, radeon, nouveau), disable KMS by adding nomodeset to the kernel boot command line. To set this permanently using Grub 2, the default boot loader, add it to the GRUB_CMDLINE_LINUX_DEFAULT kernel default load options line in your /etc/default/grub text file as root and running the terminal command 

sudo /usr/sbin/grub2-mkconfig --output=/boot/grub2/grub.cfg

for the changes to take effect. Else, for Grub Legacy, add it to the kernel command line in /boot/grub/menu.lst, also done as root. This option makes sure the appropriate kernel module (intel, radeon, nouveau) is loaded with modeset=0 in initrd, i.e. KMS is disabled.

In the rare cases when loading the DRM module from initrd is a general problem and unrelated to KMS, it is even possible to disable loading of the DRM module in initrd completely. For this set the NO_KMS_IN_INITRD sysconfig variable to yes via YaST, which then recreates initrd afterwards. Reboot your machine.

على إنتل بدون KMS ومع الخادم X يستعيد سواقة التشغيل fbdev (برنامج التشغيل intel فقط يدعم KMS)؛ وبدلاً من ذلك، ل GPUs الموروثة من Intel "intellegacy" (مجموعةxorg-x11-driver-video-intel-legacy ) يتوفر سواقة تشغيل، التي ما زالت تدعم UMS (إعداد وضع المستخدم). لتستخدمها، حرر /etc/X11/xorg.conf.d/50-device.conf وغير إدخال برنامج التشغيل إلى intellegacy.

على ATI ل GPUs الحالي يستعيد radeonhd . يتم استخدام سواقة التشغيل على NVIDIA دون KMS nv (تعتمد سواقة التشغيل nouveau KMS فقط). لاحظ أن ATI و NVIDIA GPUs الحديثة تعود إلى fbdev، إذا قمت بتحديد معلمة إقلاع النواة nomodeset.

5.2. الإقلاع مع تجاهل sysvint

بشكل افتراضي، الإقلاع باستخدام systemd. في حالة حدوث مشكلة، يمكنك محاولة التبديل مرة أخرى إلى طريقة مهملة sysvinit عن طريق الضغط على المفتاح F5 عند الإقلاع. للحصول على مزيد من المعلومات حول القيود عند الإقلاع مع sysvinit، راجع قسم 4.2, “sysvinit تم تجاهله”.

5.3. systemd: توفير معلمات بدء الخدمة

systemctl يدعم فقط المعلمات "القياسية" (انظر http://www.freedesktop.org/wiki/Software/systemd/Incompatibilities).

يمكنك تجاوز هذا السلوك الجديد باستدعاء البرنامج النصي للبدء مباشرة، على سبيل المثال:

cd /etc/init.d
./apache2 <your_parameters>

5.4. systemd: إيقاف تشغيل النظام

لإيقاف وإيقاف تشغيل النظام عند استخدام systemd، ومسألة وقف-p أو إيقاف التشغيل-h الآن على سطر الأوامر أو استخدام زر إيقاف التشغيل الذي تقدمه بيئة سطح المكتب.

ملاحظة: halt العادي سوف لن يوقف تشغيل النظام بشكل صحيح.

5.5. systemd: الاستفادة من tmpfs: /run, /var/run, /media, etc.

systemd يوصل العديد من المجلدات التي يجب أن تحتوي على بيانات متقلبة فقط، كملفات tmpfs: /run, /var/run, /var/lock, and /media هي تلك المجلدات. للحصول على معلومات، انظر http://lwn.net/Articles/436012/.

ملاحظة: لا تقم بتخزين الملفات التي يراد منها إبقاء إعادة التشغيل على قيد الحياة، في /run, /var/run, etc.

5.6. systemd: تنظيف المجلدات (/tmp and /var/tmp)

systemd يحافظ على المجلدات كما هو محدد في مجلد tmpfiles.d و /lib/systemd/system/systemd-tmpfiles-clean.timer. لمزيد من المعلومات انظرtmpfiles.d manpage.

بشكل افتراضي، ينظف systemd المجلدات tmp يومياً كما في الإعدادات /usr/lib/tmpfiles.d/tmp.conf:

d /tmp 1777 root root 10d
d /var/tmp 1777 root root 30d

ملاحظة: سيستيمد لا يحترم متغيرات sysconfig في /etc/sysconfig/cron such as TMP_DIRS_TO_CLEAR.

5.7. Auto-mounting USB Media

Gnome and Xfce now use udisks2 to automatically mount USB media under /run/media/$USER. KDE still uses udisks version 1 and mounts USB media under /media.

5.8. Specifying Partitions for Loopback Devices

With Kernel 3.4 there are two ways to have partitions for loopback devices. The first is with max_part and the second is with the -P parameter to losetup. They behave slightly differently since -P will dynamically allocate minor numbers for each device (including adding or removing them on the fly with blockdev --rereadpt). Using the max_part parameter causes each loop device to allocate that many minor numbers for each device.

So when you use max_part=8 and do not change max_loop, which defaults to 8, you are using all of the allocated minor numbers with the first device.

The solution is either to use -P or to also use max_loop.

5.9. معلومات المنطقة الزمنية في /etc/adjtime

السطر الثالث من /etc/adjtime الآن يحتوي على معلومات عن ساعة BIOS إذا كانت تعمل على التوقيت العالمي أو المنطقة الزمنية المحلية (المخزنة سابقا في HWCLOCK في /etc/sysconfig/clock).

إذا كان /etc/adjtime يحتوي على خطأ انحراف المعلومات كبير (على سبيل المثال، بعد تحديد التاريخ والوقت مع ntpdate أو يكون ntpd قيد التشغيل)، تعيين المتغير USE_ADJUST إلى "لا" في /etc/sysconfig/clock.

5.10. GNU tar Defaults to Creating POSIX-compliant Archives

GNU tar now defaults to --format=posix and create POSIX-compliant archives with PAX extended headers. Check whether your scripts and applications are compatible with this format.

The former behavior (and upstream default) can be restored by setting the environment: 

TAR_OPTIONS='--format=gnu'

or 

TAR_OPTIONS='--pax-option=delete=[ac]time*'
